Initial Foothold

I first attempted to use CVE-2011-2523 against the vsftpd service, but the exploit was unsuccessful. Anonymous FTP login was allowed, but there were no useful files in the directory.

After enumerating SMB, I found a share named tmp with read/write access. I was able to connect anonymously using smbclient, but did not find anything useful inside the share.

Privilege Escalation

I used Metasploit to exploit CVE-2007-2447 against the vulnerable Samba service. The exploit successfully provided a shell with root privileges.

Vulnerabilities / Misconfiguration

Older Samba configurations allowed a system admin to define a script such as username map script. A script like this would normally be used to help Samba translate usernames. The specific problem with older versions of Samba was that user-controlled input could be passed into the script in an unsafe way.

An attacker could craft a username that included shell metacharacters. When Samba passed that username to the script, the system could interpret part of the username as a command. This made it possible to execute commands on the target system through the vulnerable Samba service.

Defensive Takeaways

This issue can be prevented by updating Samba to a version that has patched the CVE-2007-2447 vulnerability. Additionally, only necessary services should be exposed, and SMB should not be accessible from untrusted networks.

Another mitigation is to disable the username map script option if it is not required for business operations. Organizations should also restrict access to ports 139 and 445 using firewall rules.

Lessons Learned

Overall, this was a pretty basic machine. It introduced me to searching online for vulnerabilities based on service versions and using Metasploit to exploit a known vulnerability. I also learned about the importance of properly configuring services and keeping them updated to prevent exploitation.
