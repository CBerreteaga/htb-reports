Vulnerabilities / Misconfiguration

Primary vulnerability for this system was the EternalBlue vulnerability applicable to Microsoft Windows 7 SP 1. EternalBlue is tracked as CVE-2017-0144 and targets a flaw in Microsofts SMBv1 protocol. It sends maliciously crafted data packets to a vulnerable system, triggering a severe memory error. The flaw lies in the fact that when processing certain SMB requests, Windows will try to convert FEA list structures into newer Windows NT formats. A miscalculation occurs during the conversion, causing the system to allocate lkess memory than the data requires. This leads to an overflow, which corrupts specific memory structures, granting attacker SYSTEM-level priveleges.

Defensive Takeaways

Organizations vulnerable to this exploit will need to ensure all endpoints and servers have Microsoft Security Bulletin MS17-010 installed. They should also disable SMBv1, block all inbound SMB traffic at the perimeter firewall to precent external exploitation and utilize internal firewalls and routing rules to segment critical infrastructure.

Lessons Learned

When running exploits, they may not work on the first attempt. Because EternalBlue is a kernnel heavy exploit, if it fails it could cause the SMB service to crash on the target machine, requiring a reset of target HTB machine. Exploit initially was not working because the LHOST was set to eth01 instead of tun0.
